ĐỒ ÁN TỐT NGHIỆP
ICS-GUARD
Hệ thống Giám sát & Bảo vệ Hạ tầng IoT Quan trọng
khỏi Tấn công Mạng theo Thời gian Thực
Hà Nội, 2026
1. Giới thiệu
Các hệ thống hạ tầng quan trọng — bao gồm lưới điện, hệ thống cấp nước, mạng giao thông và cơ sở công nghiệp — đang ngày càng tích hợp thiết bị IoT để tự động hoá và tăng hiệu quả vận hành. Tuy nhiên, sự kết nối này cũng mở ra bề mặt tấn công rộng lớn cho các mối đe doạ mạng.
Thực tế cho thấy các cuộc tấn công vào SCADA, smart meter và PLC công nghiệp có thể gây mất điện diện rộng, ô nhiễm nguồn nước hoặc đình trệ sản xuất. Phần lớn sự cố xảy ra vì không có hệ thống giám sát chuyên biệt đủ khả năng phát hiện sớm và phản ứng tự động.
Mục tiêu đồ án
Xây dựng hệ thống ICS-Guard — một nền tảng bảo mật IoT chuyên biệt cho hạ tầng quan trọng — có khả năng: thu thập dữ liệu từ thiết bị IoT công nghiệp, phát hiện hành vi bất thường và tấn công mạng bằng AI, tự động phản ứng và cảnh báo theo thời gian thực, hiển thị trực quan trên Security Operations Dashboard.
2. Mục tiêu đồ án
Sinh viên cần xây dựng một hệ thống có đầy đủ các năng lực sau:
Thu thập dữ liệu từ nhiều thiết bị IoT công nghiệp (cảm biến, camera, PLC, smart meter).
Truyền dữ liệu an toàn qua giao thức MQTT với mã hoá TLS.
Phân tích và phân loại sự kiện theo thời gian thực.
Phát hiện các hành vi tấn công: brute force, DDoS, port scan, man-in-the-middle.
Tự động phản ứng: cô lập thiết bị, chặn IP, leo thang cảnh báo.
Hiển thị Security Operations Dashboard với bản đồ topo và biểu đồ thống kê.
Tích hợp AI phân tích và đưa ra khuyến nghị xử lý sự cố (khuyến khích).

4. Yêu cầu Phi chức năng
Ngoài các chức năng nghiệp vụ, hệ thống phải đáp ứng các tiêu chuẩn kỹ thuật và vận hành sau:
Bảo mật: Toàn bộ API sử dụng JWT authentication; giao tiếp MQTT phải có TLS; mật khẩu lưu bằng bcrypt.
Hiệu năng: Dashboard phải phản hồi trong vòng 2 giây; pipeline xử lý sự kiện dưới 500ms end-to-end.
Giao diện: Responsive, hoạt động trên màn hình 1080p và tablet; không cần hỗ trợ IE.
API Design: Chuẩn RESTful, có Swagger/OpenAPI documentation đầy đủ với ví dụ request/response.
Logging: Ứng dụng phải ghi đầy đủ log cấu trúc (structured logging) có thể phân tích bằng chính ICS-Guard.
Mã nguồn: Lưu trên GitHub với commit có ý nghĩa, README rõ ràng và hướng dẫn cài đặt.
Triển khai: Docker Compose cho toàn bộ stack, có thể khởi động bằng một lệnh duy nhất.
Kiểm thử: Tối thiểu 60% code coverage cho các module nghiệp vụ cốt lõi.

6. Kết quả Đầu ra
Sinh viên phải nộp đủ các hạng mục sau vào thời hạn quy định:
Source Code: Toàn bộ mã nguồn trên GitHub (repository public hoặc thêm giáo viên hướng dẫn làm collaborator).
Database Script: Script khởi tạo schema và dữ liệu mẫu (seeder).
API Documentation: Swagger UI hoặc Postman Collection đầy đủ, có thể chạy được.
UML Diagrams: Use Case Diagram, Class Diagram, Sequence Diagram (ít nhất 3 luồng chính), Deployment Diagram.
Architecture Diagram: Sơ đồ kiến trúc 4 lớp (IoT → Collection → Analysis → Dashboard) với giải thích.
Testing Report: Kết quả Unit Test, Integration Test, và ít nhất 2 kịch bản kiểm thử bảo mật.
User Manual: Hướng dẫn cài đặt (Docker Compose), hướng dẫn sử dụng từng module, hướng dẫn tạo rule.
Báo cáo Đồ án: Tài liệu hoàn chỉnh theo mẫu quy định, tối thiểu 40 trang.
Slide Thuyết trình: Tối đa 20 slide, bao gồm demo live hoặc video demo 5–10 phút.
Video Demo: Ghi lại toàn bộ demo chức năng, upload lên YouTube/Drive và đính link vào báo cáo.

8. Mức độ Mở rộng
Căn cứ vào khả năng triển khai thực tế, đồ án được phân theo 3 mức độ:
Mức độ
Yêu cầu tối thiểu để đạt
Khá
Hoàn thành Module 1–4 và Module 6; Dashboard cơ bản với ít nhất 3 loại biểu đồ; Rule Engine tạo được rule đơn giản; cảnh báo Email hoạt động.
Giỏi
Hoàn thành tất cả Module 1–6 đầy đủ; Network Topology Map hiển thị thiết bị IoT thời gian thực; Device Simulator giả lập ít nhất 3 loại tấn công; Telegram Bot hoạt động; Auto Block IP khi rule kích hoạt.
Xuất sắc
Đủ tiêu chí Giỏi + ít nhất 2 trong số: AI Security Assistant phân tích sự cố; ML model phát hiện anomaly (F1-score > 0.8 trên tập test); Playbook Engine tự động hoá phản ứng; Triển khai trên Cloud (AWS/GCP/Azure) có HTTPS; MITRE ATT&CK mapping.
